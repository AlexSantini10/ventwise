# Contributing

Thanks for helping with VentWise.

## Workflow

- Check the existing GitHub issues before starting a new TODO.
- Work on a short-lived branch created from `dev`.
- Keep the change focused on a single issue.
- Open a pull request to `dev` when the task is ready.
- After the PR is merged, close the related issue and delete the branch.

## TODO Tracking

- Track every TODO as a GitHub issue.
- Keep each issue small and specific.
- Use the minimal structure:
  - `Goal`
  - `Scope`
  - `Acceptance`
- Tag the issue and assign it to `@AlexSantini10`.

## Pull Requests

- Prefer small, reviewable PRs.
- Include tests when behavior changes.
- Update documentation when user-facing behavior changes.
